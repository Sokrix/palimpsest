## palimpsest

The user maintains a palimpsest vault (Obsidian + Karpathy 3-layer LLM Wiki architecture) at:

```
{{VAULT_PATH}}
```

In skill files and below, `<vault>` is shorthand for this absolute path.

This vault is the user's persistent memory — decisions, research, projects, patterns, daily logs. Consult it when relevant. It is accessible from any Claude Code session via the global skills (`/prime`, `/ingest`, `/save`, `/compile`, `/query`, `/lint`, `/notebooklm`).

### Absolute rules (always)

1. NEVER modify, rename or move a file in `<vault>/raw/` — it is the human space, immutable
2. NEVER create an orphan note in `<vault>/wiki/` — every note has at least one incoming or outgoing wiki link
3. NEVER write in the vault outside a skill — direct edits to `<vault>/raw/` or `<vault>/wiki/` are forbidden, even when explicitly asked. Route every write through `/ingest`, `/save`, `/compile`, or `/notebooklm`.
4. NEVER delete a wiki note — archive by changing `status: archive`
5. NEVER invent information that is absent from the vault — flag when data is missing

### 3-layer architecture (Karpathy LLM Wiki)

| Layer            | Path                              | Owner | Rule                                                |
| ---------------- | --------------------------------- | ----- | --------------------------------------------------- |
| Layer 1 — Raw    | `<vault>/raw/`                    | Human | Immutable. External artifacts (clippings, docs, notes). |
| Layer 2 — Wiki   | `<vault>/wiki/`                   | LLM   | Compiled. Concepts, syntheses, index, log.          |
| Layer 3 — Schema | `~/.claude/CLAUDE.md` (this file) | Human | Rules, conventions, vault path.                     |

### How it works

- **`<vault>/raw/`** — the human dumps external content here (articles, PDFs, notes). The LLM reads but NEVER touches.
- **`<vault>/wiki/`** — compiled knowledge: structured notes, index, log. The LLM is responsible for quality.
- **`<vault>/wiki/index.md`** — steering panel. The LLM reads this FIRST to navigate the wiki without scanning every folder.
- **`<vault>/wiki/log.md`** — append-only chronological journal. Every operation adds an entry.

### Available operations

| Command       | Role                                          | When to use it                                                                  |
| ------------- | --------------------------------------------- | ------------------------------------------------------------------------------- |
| `/prime`      | Load vault context at the start of a session  | At the start of any session that will work with the vault                       |
| `/ingest`     | Compile raw/ → wiki/                          | After adding files to `<vault>/raw/`                                            |
| `/save`       | Save the session (light)                      | End of a light session — daily note only                                        |
| `/compile`    | Save the session (deep, superset of `/save`)  | End of a deep session — topical notes in Context/Intelligence/Resources + daily |
| `/query`      | Deep search in the wiki                       | To find information in the vault                                                |
| `/lint`       | Health-check of the vault                     | Periodically (1x/week recommended)                                              |
| `/notebooklm` | Vault → NotebookLM → multimedia deliverable   | To generate podcasts, mindmaps, guides from the wiki                            |

At the end of a session, run **either** `/save` **or** `/compile` — never both. Pick `/compile` when the session produced durable knowledge worth canonicalizing into topical notes. Pick `/save` for checkpoints with nothing substantive to compile.

### Obsidian conventions

- **Wiki links**: `[[Note name]]` for any internal link
- **Embeds**: `![[Note]]` to include content
- **Mandatory YAML frontmatter** on every wiki note:

```yaml
---
date: YYYY-MM-DD
tags: []
type: note | context | research | resource | daily
status: active | archive
---
```

### wiki/ structure

| Folder                       | Content                                             |
| ---------------------------- | --------------------------------------------------- |
| `<vault>/wiki/Context/`      | Context notes: profile, objectives, projects       |
| `<vault>/wiki/Intelligence/` | Decisions, research, analyses, benchmarks          |
| `<vault>/wiki/Resources/`    | Reusable templates, patterns, snippets             |
| `<vault>/wiki/Daily/`        | Daily journal auto-generated by `/save` or `/compile` |

These folders are created as needed. NEVER create an empty folder.

### Working from outside the vault

The skills work from any Claude Code session via the absolute paths above. You do not need `cd` into the vault directory to invoke them.

If a session in another workspace produces wiki-worthy content (a decision, a pattern, an analysis), use `/compile` to canonicalize it directly — no need to stage in `<vault>/raw/` first. `<vault>/raw/` is reserved for external artifacts (articles, PDFs, notes), not session output.
